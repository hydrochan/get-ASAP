    TDD RED-GREEN으로 main.py 구현. 전체 흐름 (per D-01):

    1. **argparse 설정 (per D-02):**
       - `--dry-run`: Notion 저장 + 라벨 마킹 스킵, 콘솔에 결과 출력
       - `--verbose`: logging DEBUG 레벨 활성화

    2. **setup_logging(verbose: bool) 함수:**
       - logging.basicConfig 대신 수동 설정 (파일+콘솔 동시 출력)
       - 파일 핸들러: `logs/get-asap.log`, RotatingFileHandler(maxBytes=5MB, backupCount=3)
       - 콘솔 핸들러: StreamHandler(sys.stdout)
       - 포맷: `"%(asctime)s [%(levelname)s] %(name)s - %(message)s"`
       - 기본 레벨: INFO, --verbose 시 DEBUG (per D-13)
       - logs/ 디렉토리 없으면 os.makedirs로 생성

    3. **run_pipeline(dry_run: bool = False) 함수:**
       순차 실행 흐름 (per D-01):
       ```
       a. publishers.json 로드 (json.load)
       b. Gmail 인증: service = get_gmail_service()
       c. 쿼리 생성: query = build_query(publishers)
       d. 상태 로드: state = load_state()
       e. 새 메일 가져오기: msg_ids = get_new_messages(service, state, query)
       f. msg_ids 비어있으면 "새 메일 없음" 로그 후 save_state(state) 호출하고 리턴
       g. 라벨 준비 (dry_run 아닐 때만): label_id = get_or_create_label(service)
       h. 파서 로드: parsers = load_parsers()
       i. 파서를 publisher 키로 매핑: parser_map = {p.publisher: p for p in parsers}
       j. 각 msg_id에 대해:
          - try/except로 감싸서 개별 메일 에러 시 스킵
          - msg = service.users().messages().get(userId="me", id=msg_id, format="full").execute()
          - headers에서 From, Subject 추출
          - sender로 publisher 키 결정 (publishers dict 순회)
          - parser_map에서 파서 찾기 (없으면 warning 로그 후 스킵)
          - body = extract_body(msg["payload"])
          - papers = parser.parse(body)
          - 각 paper에 대해:
            * journal이 비어있으면 infer_journal(sender, subject, publishers)로 채움
            * doi가 비어있으면 lookup_doi(paper.title)로 채움
          - all_papers.extend(papers)
       k. dry_run이면:
          - 각 paper의 title, doi, journal을 콘솔에 출력 (print로)
          - save_state(state) 호출
          - 리턴: {"extracted": len(all_papers), "saved": 0, "skipped": 0, "failed": 0}
       l. dry_run 아니면:
          - db_id = get_or_create_db()
          - result = save_papers(all_papers, db_id)
          - 각 처리 완료 msg_id에 mark_processed(service, msg_id, label_id) 호출
          - save_state(state)
          - 리턴: {"extracted": len(all_papers), **result}
       ```

    4. **실행 요약 로깅 (per D-12):**
       run_pipeline 결과를 INFO 레벨로 기록:
       `logger.info("완료: %d건 추출, %d건 저장, %d건 중복 스킵, %d건 실패", extracted, saved, skipped, failed)`
